Refactor the popup host so it renders coordinator-driven loading/result/error content instead of a single
hard-coded source string.

`PopupController.swift`:
- Change `show(sourceText:onDismiss:)` to accept the translation coordinator (or a view model wrapper around it) so the hosted SwiftUI view can observe request state
- Keep the existing single-panel reuse, screen placement, `.nonActivatingPanel`, `orderFrontRegardless()`, and dismiss monitors unchanged
- Preserve the existing compact popup size and quiet presentation; do NOT add stronger animation than the current subtle fade-in

`PopupView.swift`:
- Render `.loading` with the existing muted source-text treatment: `.secondary`, compact width, truncation, no spinner/shimmer/ellipsis
- Render `.result` in the same popup almost instantly with translated text in a readable primary style; no noticeable transition animation
- Render `.error` inline and keep the popup open until user dismissal; copy must stay short and matter-of-fact
- Execute translation inside the view lifecycle on macOS 15 using `translationTask` / `PopupView.translationTask`, not a stored session on a service object
- Run `TranslationAvailabilityClient.preflight(...)` before enabling live translation
- Use `source: nil` plus the fixed generic-English target for automatic detection
- Do NOT plan around or call macOS 26-only conveniences such as direct `TranslationSession(installedSource:target:)`, `session.cancel()`, or `TranslationError.notInstalled`
- Catch `CancellationError` silently when the view is torn down by dismiss or re-trigger, and still guard every success/error write with request identity
- Route non-`CancellationError` failures through `TranslationErrorMapper` before calling `translationCoordinator.fail(...)` so popup copy stays app-owned and short

The popup must remain Phase 2-quiet: same material background, same compact feel, same single surface.
Do not create a second window, download guidance view, or target-language picker here.
