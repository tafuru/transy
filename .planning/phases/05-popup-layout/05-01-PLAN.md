  <action>
Modify the `PopupText` view (currently lines 71-87) to support unlimited line wrapping and vertical scrolling:

**Changes to make:**
1. Remove `.lineLimit(4)` — allows Text to wrap naturally to any number of lines
2. Remove `.truncationMode(.tail)` — no truncation needed with unlimited wrapping
3. Wrap the Text view in `ScrollView(.vertical)` — enables vertical scrolling
4. Apply `.frame(maxHeight: 400)` to the ScrollView — caps scroll region at 400pt (content sizes naturally up to this limit)
5. Move `.background(...)` to outside ScrollView — frames the fixed viewport, not scrollable content

**Target structure:**
```swift
private struct PopupText: View {
    let text: String
    let isMuted: Bool

    var body: some View {
        ScrollView(.vertical) {
            Text(text)
                .font(.body)
                .foregroundStyle(isMuted ? .secondary : .primary)
                .multilineTextAlignment(.leading)
                .padding(.horizontal, 16)
                .padding(.vertical, 12)
                .frame(width: 380, alignment: .leading)
        }
        .frame(maxHeight: 400)
        .background(.regularMaterial, in: RoundedRectangle(cornerRadius: 14))
    }
}
```

**Key points:**
- Text now wraps to unlimited lines within the 380pt width
- ScrollView appears automatically when content exceeds 400pt height
- Short translations size compactly (no wasted space) — maxHeight is a ceiling, not a fixed size
- Background material and rounded corners frame the visible scroll region
- No other changes needed in this file (PopupView, LoadingPopupText remain unchanged)

**What NOT to change:**
- Do NOT modify `PopupController.swift` — NSPanel sizing adapts automatically
- Do NOT change fixed width (380pt) or padding values
- Do NOT add `.scrollIndicators(.visible)` — default `.automatic` is correct for macOS
- Do NOT use `List` instead of `ScrollView` — List adds unnecessary visual chrome for free-form text
