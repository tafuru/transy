**Transy/TransyApp.swift** — finalize icon choice (discretion: use `"character.bubble"` — quiet, translation-appropriate, renders as template image in both light/dark):
```swift
import SwiftUI

@main
struct TransyApp: App {
    @NSApplicationDelegateAdaptor(AppDelegate.self) private var appDelegate

    var body: some Scene {
        MenuBarExtra("Transy", systemImage: "character.bubble") {
            MenuBarView()
        }
        .menuBarExtraStyle(.menu)

        Settings {
            SettingsView()
        }
    }
}
```
Notes:
- `.menuBarExtraStyle(.menu)` is REQUIRED — without it, clicking the icon opens a floating panel (`.window` style), not a native dropdown.
- `"character.bubble"` renders as a template image automatically (dark/light adaptive, correct sizing). Per user decision: icon-only, quiet, native.
- Do NOT pass the `isInserted:` binding variant — the simple initializer is correct for Phase 1.
- Do NOT add an `AppState` environment object here yet — it's not consumed by any view in Phase 1.

**Transy/MenuBar/MenuBarView.swift** — exact implementation (no changes from stub needed, but confirm it matches):
```swift
import SwiftUI

struct MenuBarView: View {
    @Environment(\.openSettings) private var openSettings

    var body: some View {
        Button("Settings…") {
            openSettings()
        }
        .keyboardShortcut(",", modifiers: .command)

        Divider()

        Button("Quit Transy") {
            NSApplication.shared.terminate(nil)
        }
        .keyboardShortcut("q", modifiers: .command)
    }
}
```
Locked decisions honored:
- "Settings…" and "Quit" only — no header row, no status row, no version string.
- Standard Cmd+, for settings and Cmd+Q for quit (per research: these are native conventions).
- Divider between the two items (per research pattern and macOS convention).
