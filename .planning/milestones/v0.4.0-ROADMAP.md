# Milestone v0.4.0: DevOps & Improvements

**Status:** ✅ SHIPPED 2026-04-04
**Phases:** 10–13
**Total Plans:** 6

## Overview

Established the full CI/CD pipeline and automated release workflow, replaced the permission-requiring Double ⌘C trigger with permission-free clipboard monitoring, and replaced manual System Settings guidance for translation model downloads with the framework's built-in download prompt. Also cleaned up the translation pipeline by removing the redundant preflight language detection.

## Phases

### Phase 10: CI Pipeline

**Goal**: PRs to main are automatically validated for code style and build correctness
**Depends on**: Nothing (first phase of v0.4.0)
**Plans**: 2 plans

Plans:

- [x] 10-01: Lint configuration (.swiftlint.yml, .swiftformat) and code compliance fixes
- [x] 10-02: GitHub Actions CI workflow with lint and build-and-test jobs

**Details:**
- SwiftLint with 150-char line limit and 17 opt-in rules
- SwiftFormat with `wrapMultilineStatementBraces` and `modifierOrder` disabled to avoid SwiftLint conflicts
- Two parallel jobs (lint, build-and-test) with concurrency groups to cancel stale runs
- `fetch-depth: 0` only on build-and-test for git describe version script
- `brew install swiftlint swiftformat` required explicitly on macos-15 runners

### Phase 11: Release Automation

**Goal**: Creating a GitHub Release automatically builds a DMG and uploads it as a release asset
**Depends on**: Phase 10 (shares XcodeGen/xcodebuild patterns)
**Plans**: 1 plan

Plans:

- [x] 11-01: Release workflow and release notes config (tag trigger → Release build → DMG → GitHub Release)

**Details:**
- Tag push trigger (`on: push: tags: [v*]`)
- create-dmg via Homebrew for drag-to-Applications DMG layout; exit code 2 tolerated (warnings only)
- Auto-generated release notes via `gh release create --generate-notes` with PR categorization from `.github/release.yml`
- xcconfig approach for version: `postBuildScript` writes git tag to `Config/Version.xcconfig`

### Phase 12: Clipboard Monitoring

**Goal**: Users can translate copied text without any permission requirements — clipboard monitoring replaces Double ⌘C as the sole, always-on trigger
**Depends on**: Nothing (independent app feature)
**Plans**: 2 plans

Plans:

- [x] 12-01: Create ClipboardMonitor with polling timer, content filtering, and unit tests
- [x] 12-02: Delete legacy trigger/permission code, wire ClipboardMonitor into AppDelegate

**Details:**
- `NSPasteboard.general.changeCount` polling at 500ms intervals
- Concealed type (`com.apple.is-password-item`) and transient type filtering
- Self-write tracking (`lastWrittenContent`) prevents re-trigger loops
- Accessibility permission requirement completely eliminated
- Mode picker in Settings removed — clipboard monitoring is the only trigger mode
- `ClipboardMonitor` is `@MainActor` with full unit tests (18 test cases across 3 suites)

### Phase 13: Translation Download UI

**Goal**: Missing translation models are handled by the framework's built-in download prompt instead of manual System Settings navigation
**Depends on**: Nothing (independent app feature)
**Plans**: 1 plan

Plans:

- [x] 13-01: Remove .missingModel short-circuit, delete guidance infrastructure, let framework handle model downloads

**Details:**
- `LanguageAvailability.Status.supported` now maps to `.ready` — `session.translate()` proceeds unconditionally
- Framework shows its own native download prompt automatically for missing models
- `TranslationModelGuidance.swift`, `TranslationModelGuidanceTests.swift`, all `missingModelContext`/`guidanceState`/`openSystemSettings` code deleted
- `SettingsStore` and `GeneralSettingsView` cleaned of model guidance state
- Performance improvement: removed redundant preflight `LanguageAvailability.status()` call before every translation (was causing double language detection ML inference); `TranslationErrorMapper` covers all error cases

## Milestone Summary

**Key Decisions:**

- Phase 10, D-01: Strict SwiftLint with 150-char line limit and 17 opt-in rules
- Phase 10, D-05: SwiftFormat disables `wrapMultilineStatementBraces`/`modifierOrder` to avoid SwiftLint `opening_brace`/`modifier_order` conflicts
- Phase 11, D-01: Tag push trigger (`on: push: tags: [v*]`) for release workflow
- Phase 11, D-03: create-dmg via Homebrew with exit code 2 tolerance
- Phase 11, D-05: Auto-generated release notes with PR categorization in `.github/release.yml`
- Phase 12: Clipboard monitoring replaces Double ⌘C entirely (no mode picker); `NSPasteboard.general.changeCount` polling at 500ms
- Phase 13, D-07: `TranslationSession.cancel()` is macOS 26+ only; `configuration.invalidate()` is the correct macOS 15 cancellation approach

**Issues Resolved:**

- Double ⌘C required Accessibility permission — replaced with permission-free clipboard monitoring
- Manual "Open Language & Region" guidance was unreliable and confusing — replaced with framework-native download UI
- Redundant preflight ML inference (double language detection) on every translation — eliminated
- No CI: code style and build correctness not automatically validated — full CI pipeline added
- No automated releases: DMG had to be built manually — automated with tag trigger

**Issues Deferred:**

- Translation framework cancellation latency (re-trigger after dismiss on long translations) — known limitation of `configuration.invalidate()` on macOS 15; needs macOS 26 to use `cancel()`
- Code signing and notarization — deferred to post-1.0 for broader distribution
- Japanese↔German translation not supported — Apple Translation framework limitation, not a Transy bug

**Technical Debt Incurred:**

- None significant — Phase 13 removed more debt (guidance infrastructure) than it added
