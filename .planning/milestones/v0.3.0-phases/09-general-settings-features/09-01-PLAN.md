**Step 2 — GeneralSettingsView.swift:** Make these changes:

1. Add `import ServiceManagement` at the top (alongside existing `import SwiftUI` and `import Translation`).

2. Add a `@State private var launchAtLogin: Bool = false` property alongside the existing `@State` properties.

3. Inside the `Form`, add a new `Section("General")` **above** the existing `Section("Translation")`. Inside it, place a single `Toggle`:

```swift
Section("General") {
    Toggle("Launch at Login", isOn: $launchAtLogin)
        .onChange(of: launchAtLogin) { _, newValue in
            if newValue {
                try? SMAppService.mainApp.register()
            } else {
                try? SMAppService.mainApp.unregister()
            }
            // Re-read actual state in case register/unregister failed
            launchAtLogin = SMAppService.mainApp.status == .enabled
        }
}
```

4. In the existing `.task` modifier block, add this line at the top (before the language loading) to initialize the toggle from actual system state:

```swift
launchAtLogin = SMAppService.mainApp.status == .enabled
```

This ensures the toggle always reflects reality — if the user changes login items in System Settings, reopening the Settings window shows the current state. The `try?` pattern silently handles any registration errors, and the re-read after the call ensures the toggle never lies about the state.

Do NOT use UserDefaults for this setting. Do NOT add any alert or error UI for failed register/unregister. The toggle simply reflects what the system reports.
