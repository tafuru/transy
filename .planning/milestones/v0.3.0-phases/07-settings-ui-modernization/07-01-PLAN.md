**1. Create `Transy/Settings/GeneralSettingsView.swift`:**

Extract the existing SettingsView body content into a new GeneralSettingsView. This view:
- Accepts `settingsStore: SettingsStore` as init parameter (same injection pattern)
- Has `@State` properties: `supportedLanguages`, `guidanceState`, `selectedLanguageID` (moved from SettingsView)
- Uses SwiftUI `Form` as the top-level container for macOS System Settings-style bordered groups
- Contains a single `Section("Translation")` wrapping:
  - `Picker("Target Language", selection: $selectedLanguageID)` — keep existing Picker logic exactly as-is (the `ForEach`, `.labelsHidden()`, `.disabled()`, `.onChange()`)
  - Conditional model guidance block (the `if guidanceState != .none` block with generic/pairSpecific switch) — move inside the same Section, below the Picker
- Keeps the `.task` modifier for loading supported languages, reconciling, and checking guidance state
- Keeps both `.onChange` modifiers for `settingsStore.targetLanguage` and `settingsStore.missingModelContext`
- Keeps the `reconcileSelectedLanguage()` and `openSystemSettings()` private methods
- Do NOT add `.padding()` — Form provides its own padding
- Do NOT set `.frame(width:)` — the parent SettingsView handles sizing

**2. Create `Transy/Settings/AboutSettingsView.swift`:**

New informational About tab view:
- No parameters needed (reads from Bundle)
- Uses `Form` with a single `Section` for consistent grouped style
- Content within the section, centered:
  - App name: "Transy" in `.title2` font weight `.bold`
  - Version: read from `Bundle.main.infoDictionary?["CFBundleShortVersionString"]` — display as "Version X.Y.Z" in `.subheadline` `.secondary` style
  - Description: "A lightweight macOS menu bar translator" in `.body` `.secondary` style
  - GitHub link: `Link("GitHub Repository", destination: URL(string: "https://github.com/tafuru/transy")!)` styled appropriately
- Layout should feel like a standard macOS About page — clean, centered, minimal

**3. Rewrite `Transy/Settings/SettingsView.swift`:**

Replace the entire body with a TabView container:
```swift
struct SettingsView: View {
    let settingsStore: SettingsStore

    var body: some View {
        TabView {
            GeneralSettingsView(settingsStore: settingsStore)
                .tabItem {
                    Label("General", systemImage: "gearshape")
                }

            AboutSettingsView()
                .tabItem {
                    Label("About", systemImage: "info.circle")
                }
        }
        .frame(width: 420, height: 260)
    }
}
```
- Remove all `@State` properties (moved to GeneralSettingsView)
- Remove all `.task`, `.onChange` modifiers (moved to GeneralSettingsView)
- Remove `reconcileSelectedLanguage()` and `openSystemSettings()` methods (moved to GeneralSettingsView)
- Keep the `settingsStore` parameter — it passes through to GeneralSettingsView
- Use `.frame(width: 420, height: 260)` as starting size — adjust if content overflows
- SF Symbol icons: `gearshape` for General, `info.circle` for About (per Claude's discretion)
- TransyApp.swift remains unchanged — `Settings { SettingsView(settingsStore: ...) }` still works

**Important:** After creating the new files, run `xcodegen generate` to regenerate the Xcode project so it picks up the new Swift files. The project.yml `sources: [{ path: Transy }]` glob will auto-include them.
